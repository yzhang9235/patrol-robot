# 巡检监控 网页控制台

给运维人员用的网页界面，不用碰终端，点按钮就能完成：
- 切换/新增站点对应的厂商型号
- 在网页上直接点选画面框出面板位置、每颗LED位置(代替原来的cv2弹窗点选)
- 启动/停止巡检
- 看运行日志、最近报警记录，下载报警录像

存储数据跟你之前手动标定(`monitor_with_rules.py --calibrate`)、
Grounding DINO标定用的是**同一份文件**(`knowledge/<vendor>_<model>.json`、
`config/runtime_config.json`)，三种标定方式随便混用，不冲突。

---

## 1. 目录结构要求

跟之前的`grounding_dino_calibration`一样，这个文件夹要放在原项目文件夹下面一层：

```
巡检/
  monitor_with_rules.py      <- 需要先打补丁，见下面第2步
  led_knowledge_lookup.py
  config_manager.py
  knowledge/
  config/
  web_control_panel/           <- 这个文件夹
    app.py
    monitor_with_rules.patch.md
    templates/
      index.html
```

---

## 2. 先给 monitor_with_rules.py 打个小补丁

打开同目录下的 `monitor_with_rules.patch.md`，按里面写的三处小改动，
把 `巡检/monitor_with_rules.py` 改一下（都是"加几行"，不会改变你现在
直接在终端跑的任何行为，只有网页后台启动它的时候才会用到这些新增的开关）。

改完确认一下：终端直接跑 `python3 monitor_with_rules.py` 应该还是跟以前
完全一样，能正常弹窗、正常巡检。

## 3. 安装依赖

```bash
cd 巡检/web_control_panel
pip3 install flask opencv-python
```

(如果之前已经装过opencv-python，这行会很快跳过)

## 4. 运行

```bash
cd 巡检/web_control_panel
python3 app.py
```

看到这样的输出就说明启动成功了：

```
项目目录: /Users/xxx/Desktop/巡检
摄像头地址: rtsp://...
网页控制台启动: http://localhost:5000
```

**在这台电脑上**，浏览器打开：

```
http://localhost:5000
```

**如果想让运维人员用另一台电脑/手机访问**（跟这台电脑在同一个局域网内），
需要把 `localhost` 换成这台跑`app.py`的电脑的局域网IP，比如：

```
http://192.168.1.50:5000
```

怎么查这台电脑的局域网IP：Mac上 系统设置 → 网络 → 看到的IP地址就是。

## 5. 页面怎么用

### 巡检运行(默认打开的页面)
- **启动巡检** / **停止巡检** 按钮
- 下面是运行日志，每3秒自动刷新

### 站点与型号
- 表格显示现在配置了哪些站点，⭐标记当前正在巡检的是哪个
- 点"设为当前"可以切换（**注意：巡检运行中不能切换，要先停止**）
- 下面的表单：填站点ID + 厂商 + 型号，为下一步标定做准备
  - 站点ID是新的：等于要新建一个站点
  - 站点ID已存在：等于要重新标定这个站点
  - 型号选已有的：会自动带出这个型号已经标过的LED位置，标定时会问你要不要复用

### 标定
1. 先在"站点与型号"填好信息，切到这个标签页
2. 点"抓取当前画面"（**注意：巡检运行中不能抓取，会跟摄像头冲突，要先停止巡检**）
3. 点"开始框选面板"，然后在图上先点左上角、再点右下角
4. 点"确认保存面板框"——如果这个型号之前标过LED，这时候会弹出确认框问你要不要载入
5. 点"添加一颗LED"，在图上点左上角→右下角，然后输入这颗LED的名字
6. 重复第5步，每颗LED都框一遍；框错了可以在下面列表里点"删除"，或者用"撤销上一个"
7. 都框完了，点"保存所有LED位置"

### 报警记录
- 表格显示最近的报警：时间、颜色、常亮/闪烁、说明书解读、录像下载链接

## 6. 常见问题

**Q: 点"抓取画面"提示"巡检运行中，无法抓取画面"**
先去"巡检运行"标签页点"停止巡检"，摄像头设备同一时间只能被一个程序占用，
标定和巡检不能同时进行。

**Q: 点"设为当前"没反应/报错**
同上，巡检运行中不能切换站点，先停止。

**Q: 运行日志区域一直是空的**
检查一下`monitor_with_rules.patch.md`里的补丁是不是真的打上了，
尤其是第3处(日志级别那段)，如果没打，日志级别还是WARNING，
很多正常运行信息(比如"已加载N颗LED的标定坐标")默认不会显示。

**Q: 想让局域网内其他电脑也能打开这个网页**
确保这台跑`app.py`的电脑防火墙没有挡住5000端口，然后用它的局域网IP访问
(见上面第4步)。这台电脑不能关机/休眠，`app.py`得一直跑着，网页才能打开。

**Q: 这个网页控制台本身要一直开着吗**
`app.py`这个进程建议保持常驻运行（跟以前直接跑`monitor_with_rules.py`一样，
可以用同样的方式，比如开机自启动/screen/tmux/supervisor常驻），
运维人员平时只需要打开浏览器访问，不需要自己去启动`app.py`。
